Build Website With Hugo and Github

January 8, 30353 · 1 min · 165 words · Me

Hugo + GitHub Page搭建个人主页#

Hugo环境#

  1. 在GitHub仓库下载最新发行包

    https://github.com/gohugoio/hugo

  2. 解压至Hugo文件夹

    将该文件夹添加到系统环境变量中

Git环境#

在Git官网下载并安装最新版Git，在终端中验证

git --version

注册并配置 GitHub

  • 注册 GitHub 账号（如果还没有）。

  • 配置 Git 本地环境：

    git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub邮箱"

创建Hugo项目#

  1. 导航至要创建网页的文件夹，在cmd中输入

    hugo new site my-website
cd my-website

  2. 选择并安装主题（以PaperMod为例）

    在my-website中运行

    git submodule add --depth=1 https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod
git submodule update --init --recursive

  3. 在config.yml中添加：

    theme: ["PaperMod"]

  4. 创建页面内容

    • 创建一个新页面：
    hugo new posts/my-first-post.md

    • 编辑 content/posts/my-first-post.md 文件，修改内容。

    • 确保 draft 设置为 false：

    draft: false

  5. 本地预览

    启动 Hugo 本地服务器：

    hugo server -D

推送到 GitHub#

  1. 登录 GitHub，创建一个新的仓库，命名为 username.github.io

  2. 进入my-website/public文件夹，初始化 Git 仓库并推送

    git init
git remote add origin https://github.com/username/username.github.io.git
git add .
git commit -m "Initial commit"
git branch -M main
git push -u origin main

部署到部署到 GitHub Pages#

  1. 生成静态文件

    hugo

  2. 将 public/ 文件夹内容推送到 GitHub

    git add .
git commit -m 'Deploy website'
git push

定期更新和维护#

  1. 更新内容

    • 修改或新增内容后：

      cd public
git add .
git commit -m "Update content"
git push

  2. 升级 Hugo 或主题

    • 更新主题：

      git submodule update --remote --merge

    • 检查 Hugo 新版本，重新安装以获取新功能。

问题及解决办法#

部署完成后网页样式加载错误#

  1. 在hugo.toml中修改

    baseURL = 'https://username.github.io'

  2. 在config.yml中添加

    params:
  assets:
    disableFingerprinting: true
